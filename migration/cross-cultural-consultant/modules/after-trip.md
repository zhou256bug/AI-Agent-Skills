# Module:回国后-复盘

## 触发

- "我刚从 X 回来" / "刚回国"
- "复盘" / "回顾这次出差"
- "为什么没谈成" / "那次谈崩了"
- "下次怎么改进"

## 工作流(两阶段)

### 阶段 1:引导提问(必须先问,不要直接分析)

用户的初始描述往往不够,先问 3-4 个引导问题:

```markdown
来复盘一下,我先问你几个问题,信息够了再分析:

1. **去了哪个国家、见了谁?**
   (国家 + 客户角色:总裁/采购/合作伙伴/政府)

2. **主要目的是什么?达成了几成?**
   (拿订单/破冰/续约/解决冲突,自评 0-100%)

3. **哪个时刻你觉得不对劲?**
   (具体时刻:某句话、某个动作、某个停顿)

4. **你当时怎么反应的?**
   (实际做的事 + 现在回头看的判断)

回答完我来做分析。
```

### 阶段 2:复盘分析

当用户回答完(或主动给了足够信息),按下面模板输出:

```markdown
## 复盘:[国家] / [场景]

### 1. 你描述的事实
[一句话复述,确认理解无误]

### 2. 文化解释(2-3 个可能原因,按可能性排序)
- **最可能**:[基于该国 Delta 最大维度 + 框架,一句话点穿]
- **次可能**:[同上]
- **不要忽略**:也可能是个人原因(对方情绪/家庭/竞争对手报价等),不要全归因到文化

### 3. 哪里做对了 ✅(1-2 条)
[具体动作 + 为什么对]

### 4. 哪里可以改 ⚠️(2-3 条具体动作)
- [具体改进点 1]
- [具体改进点 2]

### 5. 下次再去,先做这件事
[1 条最关键的提前准备,可操作级]

### 6. 数据 + 框架依据
- [D] 该国 Hofstede 关键维度引用
- [F] 1 条相关框架
- [C] 若有相关课堂案例,引用 1 条
```

## 关键规则

1. **必须先问再答**——直接分析容易过度归因到文化,实际可能是个人/商业原因
2. **2-3 个可能原因要排序**,不是并列罗列
3. **"也可能是个人原因"必须提**——避免文化决定论
4. **改进点必须是动作级**,不是"加强沟通"这种废话
5. **若用户拒绝引导问题**(说"你直接说"):用现有信息分析,但明确标注"基于有限信息推断"

## 长度

- 阶段 1(提问):150 字内
- 阶段 2(分析):800-1200 字

## 用户验证钩子

复盘末尾追问一句:
> "这次分析对得上你的真实感受吗?最准的一条是?最不准的一条是?"

用户的反馈本身就是 skill 越用越准的数据。
