# Module:出国中-谈判/合同场景

## 触发

- "谈合同" / "谈价" / "推进合作"
- "下一步怎么走" / "怎么接话"
- "客户这么说是什么意思" / "怎么回应"
- "饭桌之后" / "合作意向"

## 输出模板

```markdown
💼 **[国家] 谈合同/推合作 - 直接给答案**

**1. 现在怎么办**(一句话)
[当前阶段最该做的一个动作或一句话术]

**2. 三条原则**
- **节奏**:[快推 / 慢磨 / 等对方先开口]
- **关键人**:[谁能拍板 / 是否需要绕到老板 / 中间人(wasta/介绍人)的角色]
- **合同细节**:[哪些必须写死 / 哪些可以留口头 / 该国对模糊条款的容忍度]

**3. 推荐话术**(2-3 个可直接用的句子)
- "[具体话术 1]"
- "[具体话术 2]"

⚠️ **这一步最容易踩的坑**:[1 条该阶段中国人最常犯的错]

📊 **数据依据**:[1 句话,引用该国 PD/UA/LTO 中最相关的 1-2 个维度]
🧠 **框架依据**:[引用 1 条最相关的教授框架,1 句话]

---
→ 还有其他场景要继续问吗?(送礼 / 突发情况 / 复盘上一次接触)
```

## 关键规则

1. **现在怎么办放第一行**——用户可能在车上/电梯里/对方刚去洗手间的几分钟内看
2. **话术要可直接念**,不是"建议表达 X 意思",而是给出具体的句子
3. **数据依据 + 框架依据各一句**,不展开成段
4. **该国 UA 决定合同细致度**:
   - 高 UA(日/德/法):合同必须事无巨细,模糊=焦虑源
   - 低 UA(中/印/印尼):关系到位口头先行
5. **该国 LTO 决定节奏**:
   - 高 LTO(日/中/韩):慢磨多轮,首次见面不谈核心
   - 低 LTO(美/拉美部分):快推,首次就可以摊牌
6. **该国 PD 决定决策路径**:
   - 高 PD(沙/俄/印/中):决策权在最高级,要绕到拍板人
   - 低 PD(德/北欧):中层就能定,平等沟通

## 长度

300-500 字。
