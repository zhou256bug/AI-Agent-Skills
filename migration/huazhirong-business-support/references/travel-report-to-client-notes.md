# 出差报告 → 客户简介 + 双链交流笔记 工作流

## 触发条件

菜头发来同事（如徐伟杰/Risto）的出差报告（xlsx），要求按报告中的客户列表：
1. 创建/补充各客户的简介笔记（存 Obsidian `客户维护/客户信息/`）
2. 创建出差交流笔记，每个客户列沟通要点 + 双链指向简介笔记

## 工作流程

### Step 1：读取出差报告

xlsx 文件用 openpyxl 读取：

```python
import openpyxl
wb = openpyxl.load_workbook(path)
for name in wb.sheetnames:
    ws = wb[name]
    for row in ws.iter_rows(values_only=True):
        ...
```

**关键**：Risto 的报告按国家分 Sheet，每个 Sheet 包含客户名、拜访内容、客户简介、现有终端、工作成果、问题与解决方案。

### Step 2：提取 17 个客户的关键信息

从每个 Sheet 提取：
- **客户名** — sheet 名或正文首行
- **拜访内容** — 客户的核心诉求、产品兴趣、决策链
- **简介** — 报告中已有的公司简介段落
- **现有终端** — 当前在用品牌/型号/价格
- **进展** — 已签单/测试中/跟进中/已放弃
- **问题** — 认证延期、价格战、竞争壁垒等

### Step 3：批量创建客户简介笔记

写入 `客户维护/客户信息/{国家}-{客户名}.md`

**前置检查（2026-06-02 菜头纠正后固化）**：
1. 先 `ls` 查看 `客户维护/客户信息/` 目录，了解已有笔记的命名惯例和格式
2. 读取最完整的已有笔记（如 `巴西-Pagbank.md`），逐行确认 YAML 属性、正文结构、表格样式
3. 不自创格式，菜头会直接指出"没有参照其他笔记"
4. 国家前缀参照 `latam-country-prefix-lookup.md`

**格式（2026-06-02 菜头纠正：必须完整参照 Pagbank.md 模板，含 YAML + 合作情况 + 核心信息表格）**：

```markdown
---
区域:
  - 国家名
  - 大洲/区域名
日期: YYYY-MM-DD
客户性质:
  - 公司类型
封面: ""  # 后续补Logo图片，暂无则留空
tags:
  - 客户
  - 国家名
---
## 客户简介
**公司全称** 一句话概括公司性质、核心业务、市场地位。

#### 一、合作情况
[[双链到出差交流笔记#客户名]]
- 时间线记录合作进展

#### 二、核心信息速览

| 项目 | 详情 |
|:--|:--|
| **公司全称** | ... |
| **成立 / 起步** | ... |
| **总部** | ... |
| **监管资质** | ... |
| **核心业务** | ... |
| **关键数据** | 用户数、商户数、交易额等 |
| **现有终端** | 当前在用品牌+型号+价格 |
| **官网** | ... |
```

**命名规则**：`{国家}-{客户名}.md`（去掉公司类型如 S.A./S.A.C.，去掉空格，驼峰或连字符）

**内容结构**：
1. 公司性质/总部/成立时间/地位
2. 核心业务和数据（商户数、用户数、市占率等）
3. 现有终端（品牌+型号+价格）
4. 华智融合作进展（来自报告）

**国家前缀对照**：
| 国家 | 前缀 |
|------|------|
| 萨尔瓦多 | 萨尔瓦多- |
| 哥伦比亚 | 哥伦比亚- |
| 秘鲁 | 秘鲁- |
| 乌拉圭 | 乌拉圭- |
| 阿根廷 | 阿根廷- |
| 巴拉圭 | 巴拉圭- |

### Step 4：创建出差交流笔记

写入 `客户维护/拉美X区出差交流 YYYY-MM-DD.md`

**文件命名**：`拉美{区号}区出差交流 YYYY-MM-DD.md`（如 `拉美二区出差交流 2026-06-02.md`）

**格式**：
- frontmatter：`date`、`tags`（含 `出差交流, 拉美X区, 同事名`）
- 开头：沟通对象、出差时间、覆盖国家、双链指向出差报告文件

**每个客户的结构**：
```markdown
## 🇨🇴 哥伦比亚

### N. 客户名

[[国家-客户名]]

**进展**：一句话简述当前合作状态。

**沟通要点**：
1. 具体问题1？
2. 具体问题2？
3. 具体问题3？
```

**问题来源**：从报告的"问题与解决方案"（Step 4 四）和"工作成果"（Step 3 三）中提炼。每个客户列 2-4 个问题，优先列出需要菜头决策的。

**优先级标签**放在笔记末尾：
```markdown
## 📋 优先级总结

**🔥 紧急**
1. [[客户名]] — 说明

**⚠️ 重要**
...

**📌 关注**
...
```

### Step 5：文件归档

原始出差报告 xlsx 同时复制到：
- `newpos/中转站/`（临时存档）
- `newpos/出差报告/`（正式归档）

## Pitfalls（2026-06-02 菜头纠正后更新）

- ❌ **不要用 xlrd 读取 xlsx** — xlrd 不支持 xlsx 格式，必须用 openpyxl
- ❌ **客户简介不要只用纯文本** — 必须包含 YAML 属性、合作情况区块、核心信息表格
- ❌ **公司简介不要只依赖报告中的简介段落** — 必须补充公开搜索资料（官网、维基、Crunchbase、新闻）
- ❌ **不要遗漏封面字段** — YAML 中必须保留 `封面: ""`，暂时没图留空，后续补Logo
- ✅ 客户简介用 **write_file** 批量写入，不要一个个手动建
- ✅ 交流笔记要用双链 `[[国家-客户名]]`，确保 Obsidian 图预览生效
- ✅ 报告文件要双链到交流笔记顶部的文件引用
- ✅ 沟通要点的数量控制在每个客户 2-4 个，不要太多
- ✅ 优先级排序要明确（紧急→重要→关注），帮助菜头快速决策
- ✅ 每个国家的客户用国家 emoji 分隔（🇸🇻 🇨🇴 🇵🇪 🇺🇾 🇦🇷 🇵🇾）
- ❌ 不要修改已有客户简介文件——只新建没有的
- ✅ 客户简介的内容以 Risto 报告中的简介段落为骨架，补充公开搜索信息为血肉
