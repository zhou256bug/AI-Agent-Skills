# 李丽妃PMO邮件——研发周例会数据源

## 发件人信息

- 发件人：`lifei.li@newpostech.com`（李丽妃，项目管理部）
- 主题格式：`产品研发与技术周例会_会议纪要&项目进度汇报（YYYY年M月D日）`
- 发送时间：会后下一个工作日（如6月8日研发会→6月9日15:44发出）
- 收件人：杨总、菜头、王青虹、杨博、赵国栋、方倩倩等核心管理层+研发负责人

## 邮件内容结构

邮件正文为 20 条左右的逐条项目进展总结，每条包含：
- 项目名称（如 NEW9850、P3000、MDP610 等）
- 当前状态（如"AP板预计6月17日寄出"）
- 风险/问题（如"联调周期不足"）
- 决策（如"方案A优先，测试不通过再换方案B"）

## 邮件附件

一般包含2个附件：
1. 「产品研发与技术周例会_会议纪要_YYYYMMDD.pdf」——正式会议纪要PDF
2. 「研发技术专题会议&项目进展报告（YYYYMMDD）.xlsx」——项目进展表（含工作重点、在研项目、认证更新等sheet）

## 工作流（注入到 §9）

以邮件为主体的工作流：

```
1. email_tool.py read_match "产品研发与技术周例会" 3 → 获取邮件全文
2. 邮件正文的 20 条项目序号 = 纪要骨架
3. 飞书妙记拉取本日转录 → 提取讨论细节（非邮件内容）补充在各项目下
4. 附件 xlsx 用 calamine 引擎解析 → 验证责任人和时间表
5. 以邮件结构编排纪要，妙记内容以「讨论补充」区块放入对应项目
6. 邮件不涵盖的议题（如杨总口头指示）从妙记提取单独成节
```

## 注意事项

- 邮件中的项目编号（1/2/3...20）就是菜的编号顺序，纪要编排不要打乱
- 邮件中的数据准确性 > 妙记中的提及（妙记有口误风险）
- 责任人不自创，从邮件上下文或 xlsx「在研项目」sheet 查
- 附件 PDF 偶尔有与邮件正文不一致的内容，此时以邮件正文 + xlsx 为准
