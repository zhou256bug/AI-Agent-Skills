# 端到端案例：俄罗斯 POScenter 首次来访（2026-06-03）

## 场景

代理商带来俄罗斯 POS 代理商 POScenter（年销 3 亿+ RMB）初次来访华智融深圳办公室，洽谈切换供应商。约40分钟会议。

## 输出产物

1. 客户档案 → `客户维护/客户信息/俄罗斯-POScenter.md`
2. 会议纪要 → `客户维护/客户信息/俄罗斯-POScenter meeting 2026-06-03.md`
3. Worknote 更新 → 当天日记 💻Worknote 区块

## 执行流程

### 第一步：获取飞书妙记（客户来访有录音）

```bash
# 搜索妙记
/opt/homebrew/bin/lark-cli minutes +search --query "俄罗斯" --start "2026-06-03T08:00:00+08:00" --end "2026-06-03T23:59:59+08:00"

# 取 token → 转录全文
/opt/homebrew/bin/lark-cli api GET /open-apis/minutes/v1/minutes/{token}/transcript
# 保存到 ~/download.txt（~1000行文本）
```

### 第二步：查看客户官网（背景调研）

浏览器打开客户官网（如 pos-center.ru），了解产品线、规模。

### 第三步：查阅参考模板

用 `read_file` 读取已有客户档案（如 `墨西哥-TPV.md`）和已有 meeting 纪要格式（如 `拉美二区出差交流 2026-06-02.md`）确定格式。

参考 meeting 纪要模板 `references/meeting-note-template.md`。

### 第四步：创建客户档案

写入 `客户维护/客户信息/{国家}-{客户名}.md`：
- YAML frontmatter（区域、日期、客户性质、tags）
- 客户简介（公司全称+一句话定位）
- 核心业务
- 关键信息
- 会议纪要双链

### 第五步：创建会议纪要

写入 `客户维护/客户信息/{国家}-{客户名} meeting {YYYY-MM-DD}.md`：
- YAML frontmatter（日期、客户、来访人、参与人）
- 会议基本信息（时间、地点、形式、妙记双链）
- 会议背景（为什么来、谁带来的）
- 客户情况
- 华智融介绍
- 产品讨论（选型、认证、技术能力）
- 商务讨论（合作模式、规模、独家、价格、竞争格局）
- Next Steps 表格（事项/负责人/时间）
- 备注/注意事项

### 第六步：更新 worknote

在当天日记 💻Worknote 区块添加条目（~150字）：
```
2.俄罗斯 POScenter 客户来访：代理商带俄罗斯排名前列的 POS 代理商来访，年销超 3 亿人民币，覆盖俄罗斯前十零售银行，首期项目约 4-5 万台，初选 9830 等安卓终端，要求本周五前提交规格对照和报价，已为客户建立档案和会议纪要；
```

### 第七步：用户追加其他事项

用户说「增加 Fernando 邮件」→ 用 patch 在现有 Worknote 区块插入新条目：
```
3.Fernando Alonso 发来 Mexico 财务报告，5 月运营数据更新，已回复安排下周一墨西哥双周会做详细讨论；
```

## 关键注意事项

- **会议纪要 + 客户档案创建一次完成**：不要先创建档案再等用户问「纪要呢？」
- **不单独建 Meetings/ 目录**：会议纪要和客户档案放同一个 `客户维护/客户信息/` 目录
- **转录全文约 1000 行**：阅读全文提取核心要点，不要用 AI 摘要代替逐行确认
- **检查是否有重复条目**：中途追加时如果内容与已有条目部分重合（如来访信息+建档案），合并成一条
- **客户档案必须含双链**：客户档案末尾用 `[[客户名 meeting YYYY-MM-DD]]` 指向会议纪要，反之亦然
