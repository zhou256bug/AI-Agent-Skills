# 海外经销合同审核 + 市场调研工作流

## 输入

菜头发送合同文件 + 口头说明客户背景（国家、客户名、合作历史）。

## Step 1: 合同全文提取

- `.doc` 文件：`textutil -convert txt <file> -output /tmp/contract.txt`
- `.docx` 文件：`python3 -c "import docx; print(docx.Document('<path>').text)"`
- `.pdf` 文件：用 OCR skill 或 pymupdf 提取

## Step 2: 历史合同查找

菜头经常和同一客户续签，要求对比新旧版本。务必先查历史合同：

### 查找顺序

1. **Obsidian vault**：搜客户名（如"7 Labs"、"Sergio"、"Reyes"）→ `客户维护/客户信息/`
2. **newpos 目录**：搜 `newpos/` 下所有文件，看是否有同名客户的旧协议文件
3. **菜头发来的旧文件**：菜头发来了2018年协议文件 → 直接与当前版本对比

### 对比结构

| 条款 | 旧版（年份） | 新版 | 变化影响 |
|------|:-----------:|:----:|:--------:|
| 最低采购量 | X台/年 | Y台/年 | +Y-X台 |
| 独家区域 | N国 | N国 | 扩大/缩小 |
| 付款条件 | 100%预付/分期 | ... | 风险变化 |
| 保证金 | 有/无 | ... | 变化 |
| 管辖法律+仲裁 | 某法+某地 | ... | 变化 |
| 法律登记条款 | 有/无 | ... | **重点：是否新增了本地保护法条款（如 Law 173）** |
| 产品范围 | 型号列表 | ... | 是否变化 |

### 重点标记：新增的本地法条款

对于老客户续签，**最危险的改动是对方新增了当地保护性法律条款**（如多米尼加 Law 173、巴西代理商保护法等）。这类条款：
- 在旧合同中往往完全不存在
- 是这次新推的，不是沿用旧约
- 属于 P0 底线问题，必须删除或大幅修改

## Step 3: 三类条款评估

按以下维度标记每一条款：

### 🔴 高风险 — 必须修改

1. **独家范围的合理性与最低采购量的匹配度**
   - 独家覆盖几个国家？独家区域的市场总需求量估算是多少？
   - 最低采购量是否与独家范围匹配？（对照菜头 15% 法则）

2. **本土保护法条款（多米尼加 Law 173、巴西商业代理法等）**
   - 允许A方在本地主管机构登记合同 → 终止难度和赔偿风险剧增
   - **菜头方法论（2026-05-29 确认有效）**：接受引用当地法，但必须绑定明确的终止触发条件。最终7-Labs合同保留了Law 173，同时写入了6项just cause终止条件（未达最低采购量、逾期付款15天未补、违反竞业、违反保密、跨区域销售、虚假陈述），中方可以无条件退出。**不要直接要求删除——对方诉求是合同在本国有法律执行力，不是要绑死你。**
   - 谈判工具：保留Law 173 + 写入明确的量化的终止触发条件 + 加"本地法不影响本合同的管辖法和仲裁条款"
   - 最终底线：如果对方拒绝任何终止触发条件，才考虑删除

3. **"最优惠价格"条款**
   - 是区域最优惠还是全球最优惠？建议限缩为"区域内最优惠"
   - 价格调整权：乙方调价需提前30天通知，已下订单不受影响

4. **知识产权授权范围**
   - 是否明确给甲方使用SDK/API的授权？
   - 授权范围是否仅限于"在区域内推广销售产品之目的"？

### 🟡 中风险 — 值得注意

5. **竞业限制条款**
   - 限制谁（甲方还是乙方）？限制期多长？
   - 合理范围：甲方不能在合同期内+终止后1-2年做竞品研发

6. **质保条款**
   - 行业标准：1年免费硬件保修（不含人为损坏）
   - 是否排除了适销性默示保证？（对乙方有利）

7. **赔偿上限**
   - "不超过产品价格本身" + 排除间接损失/利润损失 → 对乙方有利

8. **终止后库存处理**
   - 合同终止后甲方手里的库存怎么办？很多合同遗漏这个

### 🟢 低风险 — 可接受

9. **付款条件**：T/T 100%预付款 → 对乙方最安全
10. **贸易术语**：FCA（工厂交货）或 FOB → 确认物流成本合理
11. **管辖法律+仲裁**：中国法律/香港法律+华南国仲/CIETAC/HKIAC → 对乙方有利
12. **保密条款**：终止后3年 → 行业标准

### 缺失项检查

- [ ] Schedule/附件是否填写了产品型号和价格？
- [ ] 品牌/商标使用授权是否明确？
- [ ] 终止后库存处理是否有条款？
- [ ] 保证金/押金条款是否存在？

## Step 4: 市场验证（可选）

菜头说"查一下这几个国家的POS需求量"时执行。

### 搜索策略
- 先搜国家名 + "POS terminals market" / "payment market" / "merchant acquiring"
- 再搜国家名 + "Verifone" / "PAX" / "Ingenico" + 年份（看谁在场）
- 再搜国家名 + "fintech" / "digital payments" + "merchant count"
- 搜Statista路透社等有数据的站点

### 估算框架
- 小岛国（人口<100万）：年需求几百台，靠旅游和零售
- 中等国家（人口500-5000万）：几千到1万台
- 用菜头15%法则：一家代理能拿市场15%就是极限
- 对比合同最低采购量是否在15%合理范围内

### 输出格式
表格形式：国家 / 年需求估算 / 市场阶段 / 竞争程度 / 15%目标量

## 交付格式

结论先行。按以下结构输出：

```
## 合同风险分析

### 🔴 必须改
1. ...

### 🟡 注意
1. ...

### 🟢 可接受

### 总结
- 对乙方有利的条款
- 需要警惕的条款
- 缺失的条款
```

## 菜头偏好（铁律）

- **15%法则**：菜头认为任何代理能拿市场15%已经是极限。输出市场估算时，先算15%目标量，再对比合同最低量
- **简洁结论先行**：不要"首先...然后...最后"的线性叙述，直接说"这个合同的最大问题是..."，再展开
- **不重复说理**：同一个判断说一次就够了，菜头一听就懂
- **核心判断独立于具体数字**：菜头对市场总量的精确数字不敏感，但对"是否合理"有本能判断。总量估算标注"≈"即可，重点放在比例对比
- **新旧合同对比的节奏**：菜头发两个版本（V1/V2）或老合同+新合同，立即做逐条差异分析。不要先给摘要再问要不要对比——直接对比，结论先行
