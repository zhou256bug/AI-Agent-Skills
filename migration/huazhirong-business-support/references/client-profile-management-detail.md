# 华智融客户档案管理

## 触发条件

菜头要求为某个客户/公司创建档案，或补充/更新现有客户信息。

## 工作流程

### Step 1：查询公开信息

搜索客户公司的公开信息：官网、新闻、行业报告、合作动态。

信息收集范围：
- 公司全称、成立时间、总部
- 主营业务和产品（特别是支付/fintech/POS相关）
- 市场地位和规模
- 近期动态（合作、融资、扩展）
- 母公司和关联方
- 潜在合作方向（仅限公开信息能推断的）

### Step 2：参照现有客户档案的格式

**关键约束**：不得自创属性名称，必须严格参照已有客户文件。

已有客户文件位于 `客户维护/客户信息/`，从这些文件可看到的规范属性：

frontmatter 必须使用的属性（参照[[孟加拉-Aamra]]和[[墨西哥-TPV]]等已有档案）：
```yaml
---
日期: YYYY-MM-DD
区域:
  - 国家名
  - 大洲/区域名
客户性质:
  - 公司类型（如：移动支付、Fintech、大型支付公司、支付科技等）
tags:
  - 客户
  - 国家名
  - 区域名
  - 其他相关标签
---
```

**不需要** `封面` 字段，除非客户有明确可用的图片。

### Step 3：联系人双链规则

- 联系人条目用**双链**指向名片笔记文件
- 格式：`姓名 — 职位 [[名片文件名|📇名片]]`
- 示例：`Ha Chien Thang — CEO [[老挝-Star_FinTech-Ha_Chien_Thang|📇名片]]`
- 不要在联系人下方重复罗列邮箱/手机号，这些信息在名片笔记里

### Step 4：写入流程

**必须委派给艾玛（emma）处理**。大为只做信息查询和结构化提取，把最终内容交给 emma 写入。

Gateway / terminal（**禁止** `delegate_task`）：
```bash
hermes -p emma chat -q "按 emma-customer-maintenance 写入客户档案：路径=…；内容=…" -Q -s emma-customer-maintenance --accept-hooks
```

CLI：
```
hermes -p emma chat -q "写入客户档案：<路径>；内容：…"
```

**禁止** `profile use` / `profile run`。

## Pitfalls

- **不自创属性**：frontmatter 只使用 `日期`/`区域`/`客户性质`/`tags`，不用 `客户公司`/`品牌`/`国家` 等自创字段——这些在已有档案里没有先例
- **联系人双链**：客户档案里只用双链指名片，不重复写联系方式
- **不要删文件**：删除任何文件之前都要两次确认（菜头的铁律）
- **写入后验证**：文件写入后必须确认文件真的存在再汇报（write_file 可能返回成功但实际没写入，特别是 iCloud 路径）
