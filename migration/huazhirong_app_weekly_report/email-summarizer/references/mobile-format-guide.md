# 移动端邮件汇报格式指南

## 设计原则

为微信等移动端优化的邮件汇报格式，核心原则：

1. **单屏显示关键信息** - 用户无需多次滚动
2. **Emoji 图标快速识别** - 🔴紧急 🟡重要 📩普通
3. **紧凑布局** - 减少空行，节省空间
4. **智能截断** - 主题/摘要长度限制
5. **时间智能显示** - 今天显示时间，其他显示日期

## 格式规范

### 标题栏（紧凑）

```
📧 邮件汇报 · 06-12 17:04
```

- 日期时间格式：`MM-DD HH:MM`
- 使用中间点 `·` 分隔

### 概览行（单行）

```
📊 共 5 封 | 🔴1 | 🟡2 | 📎3
```

- 只显示关键统计：总数、紧急数、重要数、附件数
- 使用 `|` 分隔
- 无紧急/重要邮件时不显示对应项

### 分组标题

```
🔴 紧急 (1)
───────────────
```

- 使用 emoji 图标表示优先级
- 括号内显示数量
- 分隔线使用 `─` (15 个字符)

### 邮件条目

```
• [16:40] 邮件主题 @发件人
  #标签 1 #标签 2
```

- 时间格式：今天用 `HH:MM`，其他用 `MM-DD`
- 主题限制 40 字符
- 发件人简化显示（≤6 字符才显示）
- 标签最多 3 个，紧凑显示

### 底栏

```
━━━━━━━━━━━━━━━━━━━━
✅ 已全部显示
```

或

```
━━━━━━━━━━━━━━━━━━━━
💡 还有 15 封未显示，请登录邮箱查看
```

## 长度限制

| 元素 | 限制 | 说明 |
|------|------|------|
| 主题 | 40 字符 | 超出截断 + … |
| 摘要 | 100 字符 | 超出截断 + … |
| 发件人 | 6 字符 | 超过不显示 |
| 标签 | 3 个 | 只显示前 3 个 |
| 邮件总数 | 20 封 | 超过显示提示 |
| 每组邮件 | 10 封 | 避免单组过长 |

## 优先级分组顺序

1. 🔴 紧急 (urgent)
2. 🟡 重要 (important)
3. 📩 普通 (normal)

## 标签自动识别

| 标签 | 关键词 |
|------|--------|
| #会议 | 会议，meeting, 参会，会议室 |
| #待办 | 待办，todo, 请处理，需要 |
| #回复 | 回复，reply, 请确认，请反馈 |
| #批准 | 批准，审批，approve, 审核 |
| #项目 | 项目，project, 进度，开发 |
| #告警 | 告警，alert, 警告，warning |
| #系统 | 系统，system, 备份，通知 |
| #人事 | 人事，任命，招聘，hr |
| #财务 | 财务，付款，发票，报销 |

## 脚本参数推荐

```bash
python summarize.py \
  --format mobile \
  --compact \
  --max-emails 20 \
  --max-subject-length 40 \
  --max-summary-length 100 \
  --group-by priority \
  --silent-if-empty
```

## 微信推送注意事项

1. **消息长度限制** - 微信单条消息约 2000 字符，超过会被截断
2. **换行符处理** - 使用 `\n` 而非 `\r\n`
3. **Emoji 兼容性** - 使用常见 emoji，避免生僻字符
4. **链接处理** - 微信会自动转换链接，避免过长 URL
5. **静默模式** - 无邮件时输出 `[SILENT]`，避免打扰用户
