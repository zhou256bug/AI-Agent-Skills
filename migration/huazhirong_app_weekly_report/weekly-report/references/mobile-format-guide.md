# 移动端汇报格式指南

## 设计原则

1. **紧凑布局** - 减少空行，适应手机屏幕
2. **清晰分层** - 标题、分组、内容层次分明
3. **关键信息优先** - 重点成果前置，详情可选
4. **表情符号导航** - 使用 emoji 快速识别内容类型

## 格式规范

### 标题栏
```
📋 工作周报 · 第 23 期
🏢 海外应用开发部
📅 2026.06.01-06.07
━━━━━━━━━━━━━━━━━━━━
```

### 分隔线
- 主分隔：`━` × 40（U+2501）
- 次分隔：`─` × 15（U+2500）
- 用途：视觉分组，避免内容混杂

### 表情符号语义
| Emoji | 含义 | 使用场景 |
|-------|------|----------|
| 📋 | 文档/汇报 | 标题 |
| 🏢 | 部门/组织 | 部门名称 |
| 📅 | 日期/周期 | 时间范围 |
| 👤 | 人员 | 员工姓名 |
| ⭐ | 关注/重点 | 关注列表 |
| 📊 | 统计/概览 | 数据统计 |
| 🔴 | 重点/紧急 | 重点成果 |
| ⚠️ | 注意/风险 | 需要关注项 |
| ✅ | 完成/成功 | 结尾确认 |
| 💡 | 提示/建议 | 空状态提示 |

### 内容层级

#### Level 1: 主标题
```
📋 工作周报 · 第 23 期
```

#### Level 2: 分组标题
```
⭐ 您的关注
━━━━━━━━━━━━━━━━━━━━
```

#### Level 3: 人员/项目
```
👤 林旭伟
───────────────
```

#### Level 4: 要点列表
```
• Airwallex 项目：完成 9800/9810/9830 UI 适配
• BMMB 项目：Mydebit 卡测试正常
```

## 方案对比

### 方案 A：精简摘要版
**特点：**
- 每人 3-5 行要点
- 完整覆盖所有人员
- 适合日常快速浏览

**适用场景：** 日常周报，团队规模<20 人

### 方案 B：关键成果版 ✅ 用户首选
**特点：**
- 关注列表置顶（如有配置）
- 重点成果 Top 5
- 统计概览 + 风险项
- 长度约 1/3

**适用场景：** 管理层汇报，快速掌握关键信息

## 空状态处理

### 无关注配置时
```
⭐ 您的关注
━━━━━━━━━━━━━━━━━━━━

💡 暂无关注配置

📝 如何添加关注：
直接告诉强仔：
  • 关注某人："关注周强"
  • 关注项目："关注 NEWSTORE 项目"
  • 移除关注："移除关注 XXX"
```

**设计理由：**
- 明确告知状态（暂无配置）
- 提供操作指南（如何添加）
- 示例具体命令（可直接复制）

## 长度控制

| 元素 | 最大长度 | 处理方式 |
|------|----------|----------|
| 主题行 | 40 字符 | 截断 + … |
| 摘要行 | 100 字符 | 截断 + … |
| 每人内容 | 150 字符 | 显示前 2 行 |
| 总汇报 | 2000 字符 | 限制显示人数 |

## 微信兼容性

### 测试通过
- ✅ 表情符号正常显示
- ✅ 分隔线对齐正确
- ✅ 中文无乱码
- ✅ 长度适中（无需滚动多次）

### 注意事项
- 避免使用生僻 emoji（可能显示为方块）
- 分隔线用全角字符（Unicode 制表符）
- 避免过长的单行（微信可能自动换行）
