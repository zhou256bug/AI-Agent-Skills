# 收件箱扫描 · Emoji 输出规范

面向 **老板** 手机扫读的 cron 汇报，统一使用以下标记：

| Emoji | 用途 |
|-------|------|
| 📬 | 扫描标题 |
| 📌 | 结论（第一句） |
| 🔴 | 真人未读 / 高优先级 |
| 🟡 | 系统邮件统计 |
| ⭐ | 关注人邮件 |
| 📧 | 普通真人邮件 |
| 📎 | 附件 |
| 👉 | 下一步动作 |

## 铁律

1. **无未读也必须汇报** — 固定话术「没有新邮件」，禁止 `[SILENT]` 静默
2. **有未读** — 必须先 `📌 结论` 再给分级列表
3. **关注人** — Fernando A/G、Linda 等用 `⭐` 并注明处理策略（读全文 / 下载附件）
4. **禁止委派** — 不调用其他 skill；读全文用本技能 `mail_tool.py`

## 无邮件模板

```markdown
## 📌 结论

没有新邮件。

👉 下次 cron 继续扫描。
```
