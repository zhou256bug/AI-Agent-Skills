# 法务合同推理框架

本 skill 的条款评估遵循四层推理结构。

## 第一层：分类与立场

```
输入合同 → 填路由表 → 确认我方角色（甲方/乙方/用人单位）→ 加载对应 Playbook
```

**防混铁律**：经销乙方 ≠ 采购甲方 ≠ 用人单位，三者 Playbook 不可互换。

## 第二层：续签对比（如适用）

```
有旧版？ → 是 → 先做差异表 → 标记「旧无新有」条款 → 再评条款
         → 否 → 直接进入 Tier 评估
```

续签最高优先级信号：**新增本土保护法/登记条款**。

## 第三层：Tier 评估

```
每条款 → 对照 Playbook Tier 1/2/3 → 映射 🔴/🟡/🟢 → 附 Playbook 依据 + 建议措辞
```

| Tier | 含义 | 行动 |
|------|------|------|
| 1 | Deal Breaker | 暂停签署或升级老板 |
| 2 | Should-Have | 出 redline 谈判 |
| 3 | Nice-to-Have | 可让步换取 Tier 2 改善 |

**谈判策略**：Tier 3 换 Tier 2 ✅；Tier 3 换 Tier 1 ❌。

## 第四层：输出与归档

```
结论先行（📌）→ emoji 分级清单（🔴🟡🟢）→ 缺失项（📎）→ 👉 建议下一步 → ⚗️ 免责声明 → 写入 output/
```

用户要求 📱 PDF 时：先归档 `.md`，再调用 `render_review_pdf.py`（见 `pdf-review-workflow.md`）。

## 证据标准

- 条款原文必须引用或概括（不可空泛断言）
- Playbook 依据必须标明章节（如「negotiation-playbook §1.1」）
- 不确定的法律结论标注「建议外部律师确认」
- 市场估算用「≈」，重点在比例对比（15% 法则）

## 禁止事项

- ❌ 委派外部 Agent 或其他 skill
- ❌ 输出正式法律意见书口吻（应标注为内部初审）
- ❌ 混用不同场景的 Playbook
- ❌ 续签时跳过新旧对比
- ❌ 输出本地绝对路径（如 `/Users/...`）
