# POS 支付合规速查

> PCI DSS、收单牌照、密钥管理等支付行业合规要点

## PCI DSS 责任划分

| 角色 | 典型责任 |
|------|----------|
| 华智融（设备厂商） | 硬件安全设计、固件更新、安全漏洞修复 |
| 收单机构/PSP | 交易处理、PCI 认证、密钥注入 |
| 商户/代理商 | 终端部署安全、物理安全、操作合规 |

**合同核对**：
- [ ] PCI 责任矩阵是否明确（谁负责什么）
- [ ] 密钥管理责任（注入、轮换、销毁）
- [ ] 安全漏洞通知时限
- [ ] 合规审计配合义务

**RED**：华智融承担收单机构级 PCI 合规义务且无责任上限。

## 本地收单牌照

| 市场 | 要点 |
|------|------|
| 巴西 | 须通过本地收单机构；PagBank/Stone 等合作模式 |
| 拉美一般 | 多数国家要求本地实体或持牌机构 |
| 中东 | 央行/监管牌照要求因国而异 |
| 非洲 | 移动支付牌照与电信监管交叉 |

**合同核对**：
- [ ] 是否要求华智融直接持有当地支付牌照（通常不应）
- [ ] 代理商是否有本地收单资质
- [ ] 无证经营的合规风险分配

## 密钥与加密

- 密钥注入须在安全设施（KIF/KIS）完成
- 禁止在合同中要求华智融远程注入密钥
- 固件签名与完整性校验责任划分

## 反洗钱（AML）与 KYC

- 终端销售合同通常不直接承担 AML 义务
- 若涉及商户入网或交易处理，须明确 AML 责任在收单机构

## 出口管制

- POS 终端可能含加密功能 → 出口管制分类（ECCN）
- 采购元器件时核对产地与 ECCN
- 制裁名单筛查义务

## 输出建议

H 模式输出时，按市场列出：
1. 必要牌照/认证清单
2. 合同中的责任矩阵缺口
3. 🔴/🟡 分级建议
