# 合同审核输出模板

所有模式统一使用以下结构。**结论先行**，配合 `emoji-output-guide.md` 中的 emoji 标记。

---

## 通用 Emoji 规则

| 位置 | 标记 |
|------|------|
| 文档标题 | 前缀 `⚖️` |
| 结论行 | `**📌 结论**`；极高危加 `🚨` |
| 风险段 | `### 🔴 必须改` / `### 🟡 注意` / `### 🟢 可接受` |
| 下一步 | `### 👉 建议下一步` |
| 升级律师 | `> ⚗️ **免责声明**` |
| 升级老板 | 行内 `👤` |

---

## A 模式 — 海外经销

```markdown
## ⚖️ 合同风险分析 — [客户名] / [国家] / [新签|续签]

**我方立场**：乙方（Vendor）
**📌 结论**：（一句话最大风险；极高危加 🚨）

### 🔴 必须改
（📋 条款原文 → 问题 → Playbook 依据 → 💬 建议措辞）

### 🟡 注意
...

### 🟢 可接受
...

### 🔍 续签差异（如有）
| 条款 | 旧版 | 新版 | 等级 |

### 📎 缺失项检查
- ❌ …
- ✅ …

### 👉 建议下一步
1. 💬 redline …
2. 👤 升级老板 …（如需要）
```

## 归档路径

| 模式 | Markdown | PDF（用户要求时） |
|------|----------|-------------------|
| A / E | `output/合同/审核记录/YYYY-MM-DD_对方名_合同审核.md` | 同路径 `.pdf` |
| B | `output/合同/采购/审核记录/…` | 同路径 `.pdf` |
| C | `output/合同/人事/审核记录/…` | 同路径 `.pdf` |
| F / G | `output/合同/股权/审核记录/…` | 同路径 `.pdf` |

PDF 生成见 `pdf-review-workflow.md`。

## 📱 手机 PDF

用户要求「发 PDF / 手机看」时：

```bash
python3 scripts/render_review_pdf.py \
  --title "对方名 合同审核" \
  --body-md <上表归档的.md> \
  --party "对方名" --mode A
```

生成后推送 PDF 文件。
