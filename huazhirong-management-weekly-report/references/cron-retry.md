# 定时与门禁（可选运维层，平台相关）

> **可选**:调度与门禁不属于技能核心(取邮件→四段→PDF)。它们是各平台自己的运维胶水;本文给出参考规格,便于你在 Hermes/OpenClaw/系统 cron 上自行接入。

## 调度(示例:周一四轮)

源周报邮件通常**周一下午**到达,可在周一多轮重试:

| Slot | 时间 | crontab |
|------|------|---------|
| 1 | 17:10 | `10 17 * * 1` |
| 2 | 18:10 | `10 18 * * 1` |
| 3 | 19:10 | `10 19 * * 1` |
| 4 | 20:10 | `10 20 * * 1`（最终） |

> Hermes 用 `jobs.json` 注册;系统环境用 `crontab -e`;OpenClaw 用其调度机制。具体不在本技能内固化。

## 门禁(幂等状态机，建议自建)

为"防重复生成 / 失败只通知一次 / 没邮件别打扰",建议在调度前跑一个 gate 判断,返回状态:

| 状态 | 行为 |
|------|------|
| `ALREADY_DONE` | 本期 PDF 已存在 → 静默 |
| `ALREADY_FAILED` | 本期已通知过失败 → 静默 |
| `RETRY_SILENT` | 邮件未到/无 xlsx → 静默,下一轮再试 |
| `FAIL_NOTIFY` | 凭据缺失/连接异常等 → 推送告警 |
| `RUN` | 邮件+xlsx 就绪 → 执行完整流程 |

去重可直接用技能内置函数(不依赖外部 gate):

```bash
python3 -c "import sys;sys.path.insert(0,'scripts');import weekly_report_paths as p;print(p.pdf_exists_for_period(23))"
```

状态文件路径、gate 脚本实现因平台而异,请放在你的运维目录下(不进本技能仓库)。
