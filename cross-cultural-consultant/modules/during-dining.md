# Module:出国中-餐桌场景

## 触发

- "今晚请客户吃饭" / "带客户吃什么"
- "喝什么酒" / "敬酒" / "酒桌"
- "餐桌上聊什么" / "饭局" / "宴请"
- "怎么点菜"

## 输出模板

```markdown
🍽️ **[国家] 商务饭局 - 直接给答案**

**1. 选什么**
- 餐厅类型:[当地高档 / 中餐 / 西餐 / 看场合]
- 菜系/品类:[2-3 个推荐方向]
- 避免:[宗教/文化禁忌的食物]

**2. 喝什么**
- 推荐品类:[当地主流商务酒类,只给品类原则不给品牌]
- 敬酒规则:[谁先举杯 / 杯子高度 / 干杯还是抿一口]
- 能否拒酒:[宗教/文化背景下拒酒是否会冒犯]

**3. 聊什么**
- ✅ 安全话题(3 个):[家庭/运动/天气/对方文化的赞美点等]
- ❌ 禁忌话题(2-3 个):[政治/宗教/历史敏感 + 该国特定雷区]

**4. 加分动作**(1-2 条该国独有礼仪)
- [具体动作,如日本"不要给自己倒酒""先让客户点"]

⚠️ **最容易踩的 1 个坑**:[该国饭桌上中国人最常犯的具体错误]

---
→ 想聊"饭局后怎么推进合作"吗?或者还有其他场景问题?
```

## 关键规则

1. **只给品类原则,不给具体品牌**(品牌时效性差,且容易踩雷)
2. **接力问句末尾**:提示下一步可能场景,但用"或者还有其他场景问题"留开口,不强行往合同推
3. **酒桌规则要写敬酒动作**——不是泛泛说"尊重对方",而是"客户先举杯你才能举,杯口低于对方"这种具体级
4. **宗教文化禁酒地区**(沙特/阿联酋/伊朗/巴基斯坦/马来部分场合):必须先说"不喝酒",再说替代饮品(咖啡 Gahwa / 茶 / 鲜榨果汁)

## 数据调用

读取目标国 Hofstede 数据,优先关注:
- **PD** 高 → 座次和敬酒次序严格
- **UA** 高 → 餐桌礼仪繁复,迟到=不尊重
- **集体主义** → 公筷/分餐文化差异
- **放纵/克制** → 酒精接受度

## 长度

300-500 字。一屏内能看完(用户在饭局前几分钟看)。
